Qu'est-ce qu'un Pipeline GitHub ?

Un pipeline GitHub fait référence à une série d'étapes automatisées qui sont définies pour être exécutées à chaque changement dans un dépôt GitHub. Cela fait généralement partie de GitHub Actions, le système d'intégration et de déploiement continu (CI/CD) intégré à GitHub.

Fonctionnement

Un pipeline est déclenché par un événement (comme un push, une pull request, ou un tag), et suit un ensemble d'étapes définies dans un fichier YAML dans le répertoire .github/workflows/ du dépôt.

Exemple simple de pipeline


name: CI

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3
      - name: Run a script
        run: echo "Hello from GitHub Actions!"

Ce pipeline exécute un simple script à chaque fois qu'un push est effectué sur le dépôt.
