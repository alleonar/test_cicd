Qu'est-ce qu'un Workflow GitHub ?

Un workflow GitHub est un fichier de configuration qui automatise les tâches dans un projet hébergé sur GitHub. Il fait partie de GitHub Actions, la fonctionnalité CI/CD (intégration et déploiement continu) intégrée à GitHub.

Fonctionnement

Le workflow est défini dans un fichier YAML situé dans le dossier .github/workflows/ de votre dépôt. Il peut être déclenché par des événements comme un push, une pull request, ou encore manuellement.

Exemple de workflow basique


name: Exemple de workflow

on: [push]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - name: Récupérer le code
        uses: actions/checkout@v3

      - name: Lancer un script
        run: echo "Hello GitHub Workflow!"

Ce workflow se déclenche à chaque push sur le dépôt et affiche un simple message.

Utilité

Les workflows permettent d'automatiser des tâches comme :

  • Tester du code à chaque modification
  • Déployer automatiquement une application
  • Générer de la documentation
  • Exécuter des scripts programmés
